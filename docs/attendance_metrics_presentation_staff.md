# attendance_summary_with_level_metrics_EF_insert.csv 項目説明（店舗スタッフ向け）

## スライド1: この表でわかること
- どの時間帯で「人が足りないか / 多すぎるか」を見える化した表です。
- 去年の売上傾向（時間帯構成）と、今のシフト体制を比べています。
- 目的は「売上の取りこぼしを減らす」「ムダな人員配置を減らす」ことです。

---

## スライド2: まず覚える3つの指標
1. 時間帯構成比(金額,%)
- その時間帯が、1日の売上の中でどれくらいの割合か（需要の目安）
- 例: 0.1500 = 15.00%

2. 体制シェア(人数×販売力,%)
- その時間帯に、店の体制（人数×販売力）をどれくらい配分しているか
- 例: 0.1100 = 11.00%

3. 需給ギャップ(pt)
- 計算式: 需給ギャップ = 体制シェア - 時間帯構成比
- マイナス: 体制不足（売上を取りこぼす可能性）
- プラス: 体制過多（人が余っている可能性）

---

## スライド3: 体制判定(不足/適正/過多) の見方
- 不足(高): ギャップ <= -0.050
- 不足(中): -0.050 < ギャップ <= -0.030
- 不足(低): -0.030 < ギャップ <= -0.015
- 適正: -0.015 < ギャップ < 0.015
- 過多(低): 0.015 <= ギャップ < 0.030
- 過多(中): 0.030 <= ギャップ < 0.050
- 過多(高): 0.050 <= ギャップ
- 対象外: 比較に必要な値が取れない行

補足:
- このファイルの割合は /100 形式です。
- Excelでパーセント表示にすると見やすくなります。

## スライド4: 項目一覧（左から順に）
| 項目名 | 意味 | 現場での使い方 |
|---|---|---|
| 日付 | シフト日 | いつの判断か確認 |
| 店舗略称 | 店舗の短縮名 | 画面で見やすく確認 |

| 店舗コード | 店舗ID | システム連携用 |

| 体制判定(不足/適正/過多) | 需給ギャップの判定結果 | 優先対応の目安 |
| 時間帯構成比(金額,%) | 売上需要の比率 | 需要の強い時間を把握 |
| 体制シェア(人数×販売力,%) | 体制の配分比率 | 配置の厚みを確認 |

| 担当スタッフ1〜7 | その時間のメンバー | 休憩・配置の再調整に使用 |

## スライド5: 1行の読み方（例）
例:
- 時間帯構成比(金額,%) = 0.1600（16%）
- 体制シェア(人数×販売力,%) = 0.1000（10%）
- 需給ギャップ(pt) = -0.0600（-6pt）
- 体制判定 = 不足(高)

解釈:
- 「売れやすい時間に対して体制が薄い」状態
- 対応案: 休憩を前後へ移動、応援投入、販売力の高いスタッフを当該時間へ移す

---

## スライド6: 現場アクションの基本ルール
- 不足(高)/(中): 最優先で見直し
  - 休憩の移動
  - 1名応援配置
  - 販売力の高いメンバーをピーク時間へ
- 過多(高)/(中): 配置最適化
  - 前後の不足時間へ再配分
- 適正: 現状維持

---

## スライド7: 注意点
- これは「予測の目安」であり、確定結果ではありません。
- 天候・イベント・販促など当日の要因で変動します。
- 週次で見て改善が進んでいるかを確認してください。

---

## スライド8: まとめ
- この表は「需要」と「体制」を同じ土俵で比べるためのものです。
- まずは「不足(高)/(中)」時間帯から改善すると効果が出やすいです。
- 目標は、売上機会の取りこぼしを減らし、ムダな配置も減らすことです。
